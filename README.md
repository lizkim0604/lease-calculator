# 🛒 AI 장보기 도우미 (AI Grocery Assistant)

> 메뉴 이름 하나만 입력하면, AI가 필요한 재료를 뽑고 실제로 구매 가능한 상품(상품명·가격·판매처)까지 찾아주는 웹 서비스입니다.

**Gemini API**로 자연어 메뉴를 구조화된 재료 목록으로 변환하고, **Naver 쇼핑 검색 오픈 API**로 각 재료의 실제 판매 상품을 조회하는, 두 개의 외부 오픈 API를 조합한 풀스택 미니 프로젝트입니다.

## ✨ 이 프로젝트로 보여주고 싶은 것

- **외부 오픈 API 연동 능력**: 서로 다른 두 개의 API(생성형 AI API + 상품 검색 API)를 하나의 파이프라인으로 조합
- **비정형 → 정형 데이터 변환**: LLM의 자유 응답을 JSON 스키마로 강제하고, 파싱 실패에 대한 방어 로직까지 처리
- **실전형 에러 핸들링**: API 키 누락, JSON 파싱 실패, 검색 결과 없음 등 실패 케이스를 화면에서 사용자에게 안내
- **작은 것도 완성도 있게**: 기능만 되는 수준을 넘어서 실제 서비스처럼 보이는 UI/UX

## 🧩 동작 원리

```
사용자 입력 (메뉴, 인분)
        │
        ▼
① Gemini API  ── 메뉴 → 재료 목록 JSON 생성
        │           예: "비빔밥, 2인분" → [계란, 소고기 다짐육, 고추장 ...]
        ▼
② Naver 쇼핑 검색 오픈 API ── 재료 키워드 → 실제 상품명·가격·판매몰·링크
        │
        ▼
Flask 웹 페이지에 영수증 스타일로 렌더링
```

| 구분 | 사용 기술 |

| Backend | Python, Flask |
| 외부 API | Google Gemini API, Naver 쇼핑 검색 오픈 API |

## 📋 주요 기능

- 메뉴명 + 인분 수 입력 → AI가 필요한 재료 목록을 자동 생성
- 재료별로 실제 판매 중인 상품(썸네일·상품명·가격·판매몰) 카드 노출, 클릭 시 실제 구매 페이지로 이동
- Gemini 응답이 JSON 형식을 벗어나거나(마크다운 코드블록 등), 특정 재료의 검색 결과가 없는 경우에도 서비스가 죽지 않고 화면에 원인을 안내
- API 키 미설정 등 설정 오류를 사용자 친화적인 문구로 표시

## 🔧 트러블슈팅: 왜 SSG 직접 스크래핑 대신 네이버 쇼핑 API인가

처음에는 SSG 검색 결과 페이지를 직접 크롤링해 실제 상품 정보를 가져오려 했지만,
SSG는 `robots.txt`에서 자동화된 접근을 막고 있어 직접 스크래핑은 이용약관 위반 소지가 있고,
페이지 구조가 바뀌면 바로 깨지는 문제도 있습니다.

대신 **네이버 쇼핑 검색 오픈 API**(공식, 무료, 인증만 받으면 즉시 사용 가능)를 사용하면
합법적으로 실제 상품명·최저가·판매처·상품 링크를 JSON으로 받을 수 있고,
네이버 통합 쇼핑검색 특성상 SSG를 포함한 여러 쇼핑몰 상품이 함께 검색됩니다.
"오픈 API를 조합해 실제 서비스를 만드는 능력"을 보여주기에도 더 적합한 방식입니다.

## 폴더 구조

```
recipe_shopper/
├── app.py                     # Flask 라우팅
├── services/
│   ├── gemini_service.py      # Gemini API 호출 (메뉴 → 재료 JSON)
│   └── naver_service.py       # 네이버 쇼핑 API 호출 (재료 → 실제 상품)
├── templates/
│   ├── index.html             # 메뉴 입력 폼
│   └── result.html            # 장보기 결과 (영수증 스타일)
├── static/style.css
├── requirements.txt
└── .env.example
```

## 실행 방법

1. 의존성 설치
   ```bash
   pip install -r requirements.txt
   ```

2. API 키 발급
   - Gemini API 키: https://aistudio.google.com/apikey
   - 네이버 오픈 API (Client ID/Secret): https://developers.naver.com
     → "애플리케이션 등록" → 사용 API에서 **검색** 체크 → Client ID/Secret 확인
     (쇼핑 검색 API는 별도 신청 없이 "검색" API 권한에 포함되어 있습니다)

3. `.env.example`을 복사해 `.env`를 만들고 키 입력
   ```bash
   cp .env.example .env
   ```

4. 서버 실행
   ```bash
   python app.py
   ```
   브라우저에서 `http://localhost:5000` 접속

## ⚠️ 알려진 제약사항

- 네이버 쇼핑 검색 API는 무료 사용량 한도가 있습니다. 재료 개수만큼 호출되므로
  메뉴가 복잡할수록(재료 15개 등) 호출 수가 늘어난다는 점을 고려하세요.
- Gemini가 가끔 JSON 형식을 깨뜨려 응답할 수 있어 `gemini_service.py`에서
  마크다운 코드블록 제거 등 방어 로직을 넣어뒀습니다. 그래도 파싱 실패 시
  에러 메시지를 화면에 그대로 보여줍니다 (재시도 로직은 추후 개선 포인트).
- 현재는 개발 서버(`flask run` / `python app.py`)로만 실행되며, 프로덕션 배포는 되어있지 않습니다.
